# 🔵 Cappello LAVORO  -  Prompt Template

Prompt pronti all'uso per il contesto lavorativo. Copia, adatta, usa.

---

### Brief giornaliero
*Ogni mattina per partire con le priorità chiare.*

```
Buongiorno. Analizza le mie email non lette e il calendario di oggi.
Dammi:
1. Le 3 priorità con stima del tempo
2. Thread che aspettano risposta da più di 24h
3. Conflitti o sovrapposizioni nell'agenda
4. Un blocco consigliato per lavoro profondo (almeno 90 minuti liberi)
```

---

### Review codice / architettura
*Per ottenere feedback strutturato su qualsiasi soluzione tecnica.*

```
Rivedi questo [codice / design / architettura]. Contesto: [piattaforma, linguaggio, vincoli].

Dammi:
- Problemi trovati e perché sono problemi
- Alternative che non ho considerato
- Una domanda di controllo per verificare la mia comprensione

⚠️ Segnala esplicitamente se ci sono rischi di sicurezza o costi imprevisti.
```

---

### Email professionale
*Per bozze di email a clienti, stakeholder o colleghi.*

```
Scrivi una bozza email a [ruolo del destinatario, es. "CTO di un cliente retail"] su [argomento].
Tono: professionale ma diretto. Massimo 150 parole.
Contesto aggiuntivo: [eventuali dettagli rilevanti]

⚠️ DA VERIFICARE  -  non inviare senza mia revisione.
```

---

### PoC / MVP veloce
*Per strutturare rapidamente una prova di concetto.*

```
Voglio un PoC per [obiettivo] su [piattaforma]. Stack: [tecnologie].

Dammi:
1. Architettura minima funzionante
2. Codice scheletro con i punti critici
3. Cosa togliere per velocizzare al massimo
4. Rischi tecnici principali

⚠️ Da validare prima del deploy.
```

---

### Preparazione riunione / presentazione
*Per arrivare preparato a meeting con clienti o stakeholder.*

```
Devo presentare [argomento] a [audience: es. "board non tecnico", "team di sviluppo", "cliente enterprise"].
Durata: [X minuti].

Crea:
1. Struttura della presentazione (sezioni + tempo stimato per ognuna)
2. I 3 messaggi chiave che devono ricordare
3. Le 2-3 domande difficili che probabilmente mi faranno
4. Una risposta preparata per ognuna
```

---

### Analisi di un problema tecnico
*Per scomporre un problema complesso in modo strutturato.*

```
Ho questo problema: [descrizione del problema].
Contesto: [sistema coinvolto, vincoli, cosa ho già provato].

Dammi:
1. Le cause più probabili ordinate per probabilità
2. Come verificare velocemente quale causa è quella reale
3. La soluzione per ognuna
4. Come prevenirlo in futuro
```

---

### Feedback su documento / slide
*Per review di output scritti prima di condividerli.*

```
Rivedi questo [documento / deck / report]. Destinatari: [audience].
Obiettivo del documento: [cosa deve ottenere chi lo legge].

Dammi:
- Messaggi poco chiari o ambigui
- Punti dove l'audience potrebbe perdersi
- Suggerimenti di struttura o taglio
- Cosa togliere per renderlo più efficace

⚠️ DA VERIFICARE prima di condividere.
```
